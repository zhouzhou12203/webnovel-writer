## 审查要求
直接写出你的审查意见，像编辑批注一样，简洁指出问题和建议。
不需要 JSON 格式，不需要评分，直接用自然语言写。
控制在 300 字以内。

重点关注四类问题：
1. 与大纲/设定/角色档案的矛盾
2. 与上章衔接是否顺畅
3. 文笔和节奏的明显问题
4. 商业留存问题：开头是否有钩子，前段是否有兑现，章末是否有继续读的断点

如果存在明显影响追读的缺陷，优先直接指出，不要被“没写错”掩盖。
以下情况应明确点名：
- 前 800-1200 字只有铺垫、没有有效推进
- 主角整章太被动，没有新增筹码
- 爽点目标已写进大纲，但正文兑现太弱
- 章末没有明确的下一步冲突、收益或揭晓

没有问题就说"本章质量良好，无明显问题"。
